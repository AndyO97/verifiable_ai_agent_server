================================================================================
OLLAMA SETUP & CONFIGURATION GUIDE (Alternative LLM Provider)
================================================================================

This guide covers setting up Ollama as an ALTERNATIVE local LLM provider.

⚠️  NOTE: OpenRouter.ai is the PRIMARY/RECOMMENDED LLM provider for this project
(see README.md). Ollama setup is optional for users who prefer local inference.

Quick setup guide for running Ollama with the Verifiable AI Agent Server.

================================================================================
SECTION 1: SYSTEM REQUIREMENTS
================================================================================

MINIMUM REQUIREMENTS
- CPU: Multi-core processor (4+ cores)
- RAM: 8GB minimum (16GB+ recommended)
- Disk Space: 20-50GB free
- GPU: Optional (NVIDIA CUDA or Apple Metal for acceleration)
- Internet: Required for model download (one-time)

SUPPORTED SYSTEMS
- Windows 10/11
- macOS 11+
- Linux (Ubuntu 20.04+, Fedora 35+)

================================================================================
SECTION 2: INSTALLING OLLAMA
================================================================================

WINDOWS & macOS
===============
1. Visit https://ollama.ai/download
2. Download the installer for your OS
3. Run the installer and follow the prompts
4. Restart your computer
5. Verify installation:
   
   $ ollama --version

LINUX (Ubuntu/Debian)
=====================
1. Open terminal and run:
   
   $ curl https://ollama.ai/install.sh | sh

2. Verify installation:
   
   $ ollama --version

================================================================================
SECTION 3: STARTING OLLAMA SERVER
================================================================================

WINDOWS & macOS
===============
Ollama runs automatically in the background after installation.
- Look for Ollama icon in system tray (Windows) or menu bar (macOS)
- Server listens on http://localhost:11434

To start manually from command line:
   $ ollama serve

LINUX
=====
Start the service:
   $ sudo systemctl start ollama

Verify it's running:
   $ sudo systemctl status ollama

View logs:
   $ sudo journalctl -u ollama -f

================================================================================
SECTION 4: CHOOSING & PULLING A MODEL
================================================================================

RECOMMENDED MODELS FOR THIS PROJECT

Small & Fast (Recommended for testing):
- mistral (4.1GB) ⭐ BEST CHOICE
  Fast inference, good quality, best for tool calling

- neural-chat (4.1GB)
  Optimized for conversational tasks

Medium Size (Balanced):
- llama2 (3.8GB)
  General purpose, widely tested

PULLING A MODEL
===============

Step 1: Verify Ollama is running
- Windows/macOS: Check system tray/menu bar for Ollama icon
- Linux: Run: sudo systemctl status ollama

Step 2: Open terminal/PowerShell in your project directory:

   $ cd "path/to/Project v2"

Step 3: Download the model:

   $ ollama pull mistral

   This may take 5-30 minutes depending on internet speed.
   Progress bar will show download status.

Step 4: Verify model is loaded:

   $ ollama list

   Expected output:
   NAME         ID              SIZE    MODIFIED
   mistral      0bfb2f4da2      3.8 GB  2 minutes ago

================================================================================
SECTION 5: TESTING OLLAMA CONNECTION
================================================================================

VERIFY SETUP WITH LANGFUSE INTEGRATION
=======================================

To test Ollama with the project (with Langfuse tracing):

1. Ensure Langfuse is running (see LANGFUSE_SETUP_GUIDE.md)
2. Set environment variable to use Ollama:
   
   $env:OLLAMA_BASE_URL="http://localhost:11434"
   $env:OLLAMA_MODEL="mistral"

3. Run the demo with Ollama:
   
   python real_prompt_demo.py

This will use Ollama for LLM inference while maintaining Langfuse tracing.

ALTERNATIVE: Force Ollama when OpenRouter key exists:
   
   $env:USE_OLLAMA="1"; python real_prompt_demo.py

TROUBLESHOOTING COMMON ISSUES
==============================

Issue: "Cannot connect to http://localhost:11434"
Solution:
1. Start Ollama (see Section 3)
2. Wait 10 seconds for server to start
3. Verify: curl http://localhost:11434/api/tags

Issue: "Model 'mistral' not found"
Solution:
1. Run: ollama pull mistral
2. Wait for download to complete
3. Run diagnostics again

Issue: "Out of memory" error
Solution:
1. Check available RAM: 8GB minimum required
2. Close other applications
3. Try smaller model: ollama pull neural-chat

================================================================================
COMPARISON: OPENROUTER vs OLLAMA
================================================================================

OpenRouter (PRIMARY - Recommended)
- Free API key: https://openrouter.ai/keys
- Cloud-based: No local setup needed
- Model selection: 100+ free and paid models
- Setup time: ~2 minutes
- Best for: Quick setup, multiple models, production

Ollama (ALTERNATIVE - Optional)
- Free: Fully local inference
- Local-based: Requires downloads + resources
- Model selection: Community models (mistral, llama2, etc.)
- Setup time: 30-60 minutes + model download
- Best for: Privacy-focused, offline work, learning
